# Sistema de Inscrição, Programação e Certificados

## Visão Geral

Este projeto consiste no desenvolvimento de um sistema web para gerenciamento do **Congresso Unificado UFLA Paraíso**, contemplando funcionalidades como inscrição de participantes, inscrição em minicursos, visualização da programação, registro de presença e emissão automática de certificados.

O sistema está sendo desenvolvido pela empresa fictícia **QuadCode**, utilizando metodologias ágeis e com foco na experiência do usuário.

---

## O Desafio

O principal desafio do projeto é desenvolver uma solução digital que substitua processos manuais e centralize todas as funcionalidades do evento em uma única plataforma.

Para isso, foram necessárias as seguintes ações:

- Refinamento dos requisitos do sistema a partir das necessidades do cliente  
- Estruturação e evolução do protótipo  
- Início do desenvolvimento do sistema  
- Planejamento da integração com outros sistemas  
- Organização do desenvolvimento por meio de sprints utilizando Scrum  

---

## Contexto

Atualmente, o gerenciamento do congresso apresenta limitações como:

- processos descentralizados  
- dificuldade no controle de participantes  
- emissão manual de certificados  
- ausência de integração entre etapas  

O sistema proposto busca automatizar esses processos, reduzir erros e melhorar a organização e o acesso às informações.

---

## Ciclo de Vida

O projeto segue uma abordagem baseada em:

- Scrum com divisão em 7 sprints  
- desenvolvimento iterativo e incremental  
- validação contínua  

Cada sprint representa uma evolução do sistema, desde a documentação e prototipação até a implementação e testes.

---

## Usuários do Sistema

Os principais usuários são:

- participantes do congresso  
- alunos de minicursos  
- palestrantes  
- organizadores do evento  

---

## Funcionalidades do Sistema

O sistema contempla as seguintes funcionalidades:

- cadastro e autenticação de usuários  
- inscrição no congresso  
- inscrição em minicursos  
- visualização da programação  
- registro de presença por meio de QR Code  
- emissão automática de certificados  

---

## Análise de Tarefas

O fluxo principal do sistema foi estruturado da seguinte forma:

1. cadastro ou login do usuário  
2. realização de inscrição  
3. participação nas atividades  
4. registro de presença  
5. emissão de certificado
